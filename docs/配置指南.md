## 基础配置

### 服务配置

```bash
# HTTP 服务端口
SERVER_PORT=8080

# GoPay 对外访问基础地址
# 用于生成支付渠道 Webhook/回调地址
# 生产环境必须填写公网可访问域名
PUBLIC_BASE_URL=https://pay.example.com

# 日志级别: debug, info, warn, error
LOG_LEVEL=info

# 日志文件路径（可选）
LOG_FILE=logs/gopay.log

# 告警 Webhook（可选，用于通知失败和对账异常）
ALERT_WEBHOOK_URL=https://your-alert-webhook
```

### 数据库配置

```bash
DB_HOST=localhost
DB_PORT=5432
DB_USER=gopay
DB_PASSWORD=your_secure_password
DB_NAME=gopay
DB_SSLMODE=disable  # 生产环境建议使用 require
```

**初始化数据库**

```bash
# 创建数据库
createdb -U postgres gopay

# 运行迁移
go run cmd/migrate/main.go up

# 或使用 Docker
docker-compose up -d postgres
```

## 支付宝配置

### 1. 创建应用

访问 [支付宝开放平台](https://open.alipay.com/)：

1. 登录开发者账号
2. 进入「控制台」→「网页/移动应用」
3. 点击「创建应用」
4. 填写应用信息并提交审核

### 2. 生成应用密钥

**生成 RSA2 密钥对**

```bash
# 生成私钥
openssl genrsa -out alipay_private_key.pem 2048

# 生成公钥
openssl rsa -in alipay_private_key.pem -pubout -out alipay_public_key.pem

# 查看公钥内容（上传到支付宝）
cat alipay_public_key.pem
```

**上传公钥到支付宝**

1. 进入应用详情 → 「开发信息」
2. 点击「接口加签方式」→「设置」
3. 选择「公钥」模式
4. 粘贴 `alipay_public_key.pem` 的内容
5. 保存后获取「支付宝公钥」

### 3. 配置环境变量

```bash
# 应用 ID（从应用详情页获取）
ALIPAY_APP_ID=2021001234567890

# 应用私钥（完整的 PEM 格式，包含 BEGIN/END 标记）
ALIPAY_APP_PRIVATE_KEY=-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA...（省略）
-----END RSA PRIVATE KEY-----

# 支付宝公钥（从开放平台获取，不是应用公钥！）
ALIPAY_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...（省略）
-----END PUBLIC KEY-----

# 网关地址
# 正式环境
ALIPAY_GATEWAY_URL=https://openapi.alipay.com/gateway.do
# 沙箱环境（测试用）
# ALIPAY_GATEWAY_URL=https://openapi.alipaydev.com/gateway.do

# 签名类型（推荐 RSA2）
ALIPAY_SIGN_TYPE=RSA2

# 字符集
ALIPAY_CHARSET=utf-8

# 数据格式
ALIPAY_FORMAT=json
```

### 4. 配置支付方式

在应用详情中开通以下功能：

- **手机网站支付**: `alipay.trade.wap.pay`
- **电脑网站支付**: `alipay.trade.page.pay`
- **APP 支付**: `alipay.trade.app.pay`
- **当面付**: `alipay.trade.precreate`（扫码支付）

## 微信支付配置

### 1. 申请商户号

访问 [微信支付商户平台](https://pay.weixin.qq.com/)：

1. 注册并提交资质审核
2. 审核通过后获得「商户号」（mch_id）

### 2. 创建应用

**方式一：公众号/小程序**

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 获取「AppID」
3. 在「微信支付」中关联商户号

**方式二：APP 支付**

1. 登录 [微信开放平台](https://open.weixin.qq.com/)
2. 创建移动应用
3. 获取「AppID」
4. 在商户平台关联应用

### 3. 设置 API 密钥

登录商户平台：

1. 进入「账户中心」→「API 安全」
2. 点击「设置 APIv3 密钥」
3. 设置 32 位密钥（建议使用随机生成）

```bash
# 生成 32 位随机密钥
openssl rand -hex 16
```

### 4. 申请 API 证书

1. 进入「账户中心」→「API 安全」
2. 点击「申请 API 证书」
3. 下载证书工具并生成证书
4. 下载证书文件：
   - `apiclient_cert.pem`（商户证书）
   - `apiclient_key.pem`（商户私钥）

**查看证书序列号**

```bash
openssl x509 -in apiclient_cert.pem -noout -serial
```

### 5. 配置环境变量

```bash
# 商户号
WECHAT_MCH_ID=1234567890

# 应用 ID
WECHAT_APP_ID=wx1234567890abcdef

# API v3 密钥（32 字符）
WECHAT_API_V3_KEY=your32characterapiv3keyhere1234

# 商户证书序列号
WECHAT_SERIAL_NO=1234567890ABCDEF1234567890ABCDEF12345678

# 商户私钥路径
WECHAT_PRIVATE_KEY_PATH=certs/wechat/apiclient_key.pem

# 商户证书路径
WECHAT_CERT_PATH=certs/wechat/apiclient_cert.pem

# 微信支付平台证书路径（可选，会自动下载）
WECHAT_PLATFORM_CERT_PATH=certs/wechat/platform_cert.pem

# API 地址
WECHAT_API_URL=https://api.mch.weixin.qq.com

# Webhook 通知地址（生产环境必填）
WECHAT_NOTIFY_URL=https://your-domain.com/api/v1/webhook/wechat
```

### 6. 放置证书文件

```bash
# 创建证书目录
mkdir -p certs/wechat

# 复制证书文件
cp /path/to/apiclient_cert.pem certs/wechat/
cp /path/to/apiclient_key.pem certs/wechat/

# 设置权限（仅所有者可读）
chmod 600 certs/wechat/apiclient_key.pem
```

## 安全配置

### 主密钥（Master Key）

用于加密存储商户密钥，**极其重要**！

```bash
# 生成强密钥
openssl rand -base64 32

# 配置
MASTER_KEY=your-generated-master-key-here
```

**密钥轮换**

建议每 180 天轮换一次：

```bash
# 使用密钥轮换工具
go run cmd/rotate-keys/main.go \
  --old-key "old_master_key" \
  --new-key "new_master_key"
```

详见 [密钥轮换指南](密钥轮换指南.md)

### 管理后台认证

```bash
# 生成 API Key
openssl rand -base64 32

# 配置
ADMIN_API_KEY=your-generated-api-key-here

# IP 白名单（逗号分隔，支持 CIDR）
ADMIN_IP_WHITELIST=127.0.0.1,192.168.1.0/24,::1
```

**使用方法**

```bash
# 访问管理后台时添加 Header
curl -H "X-API-Key: your-api-key" http://localhost:8080/admin/orders
```

### Docker 迁移控制

```bash
# 是否在容器启动时自动运行迁移
RUN_MIGRATIONS=false  # 生产环境建议手动控制

# 迁移文件路径
MIGRATIONS_PATH=file://migrations
```

## 生产环境配置

### HTTPS 配置

```bash
# 启用 HTTPS
ENABLE_HTTPS=true

# TLS 证书路径
TLS_CERT_PATH=/path/to/cert.pem
TLS_KEY_PATH=/path/to/key.pem
```

### 限流配置

```bash
# 启用限流
RATE_LIMIT_ENABLED=true

# 每个 IP 的请求限制
RATE_LIMIT_REQUESTS=100

# 时间窗口
RATE_LIMIT_DURATION=1m
```

### 跨域配置

```bash
# 允许的来源
CORS_ALLOWED_ORIGINS=https://your-frontend.com,https://admin.your-domain.com

# 允许的方法
CORS_ALLOWED_METHODS=GET,POST,PUT,DELETE

# 允许的 Header
CORS_ALLOWED_HEADERS=Content-Type,Authorization,X-API-Key
```

### 错误追踪（可选）

```bash
# Sentry DSN
SENTRY_DSN=https://xxx@sentry.io/xxx

# 环境标识
SENTRY_ENVIRONMENT=production
```

### 对账任务配置

```bash
# 启用自动对账
RECONCILIATION_ENABLED=true

# Cron 表达式（每天凌晨 2 点执行）
RECONCILIATION_SCHEDULE=0 2 * * *
```

## 常见问题

### Q1: 支付宝公钥和应用公钥有什么区别？

- **应用公钥**: 你生成的公钥，上传到支付宝平台
- **支付宝公钥**: 支付宝平台的公钥，用于验证支付宝的签名

配置时使用的是「支付宝公钥」，不是「应用公钥」！

### Q2: 微信支付证书序列号在哪里查看？

```bash
openssl x509 -in apiclient_cert.pem -noout -serial
```

输出格式为 `serial=1234567890ABCDEF...`，取等号后面的值。

### Q3: 如何测试 Webhook 回调？

使用 [ngrok](https://ngrok.com/) 或 [localtunnel](https://localtunnel.github.io/www/) 暴露本地服务：

```bash
# 使用 ngrok
ngrok http 8080

# 将生成的 URL 配置到 WECHAT_NOTIFY_URL
WECHAT_NOTIFY_URL=https://abc123.ngrok.io/api/v1/webhook/wechat
```

### Q4: 生产环境需要修改哪些配置？

必须修改：
- `MASTER_KEY`: 使用强随机密钥
- `ADMIN_API_KEY`: 使用强随机密钥
- `DB_PASSWORD`: 使用强数据库密码
- `ALIPAY_GATEWAY_URL`: 改为正式环境
- `WECHAT_NOTIFY_URL`: 改为生产域名
- `DB_SSLMODE`: 改为 `require`

建议启用：
- HTTPS 配置
- 限流配置
- Sentry 错误追踪

### Q5: 如何备份配置？

```bash
# 备份环境变量文件（注意安全！）
cp .env .env.backup.$(date +%Y%m%d)

# 加密备份（推荐）
gpg -c .env
```
