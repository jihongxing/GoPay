# 微信支付配置指南

## 前置准备

### 1. 注册微信支付商户号

访问 [微信支付商户平台](https://pay.weixin.qq.com/) 注册商户号。

### 2. 获取必要信息

登录微信支付商户平台，获取以下信息：

- **商户号（mch_id）**：账户中心 → 商户信息
- **APIv3密钥（api_v3_key）**：账户中心 → API安全 → 设置APIv3密钥
- **商户证书（apiclient_key.pem）**：账户中心 → API安全 → 申请API证书
- **证书序列号（serial_no）**：账户中心 → API安全 → 查看证书

### 3. 配置支付产品

产品中心 → 开发配置：
- 开通 Native 支付
- 配置支付授权目录
- 配置回调地址

## 配置步骤

### 1. 准备证书文件

将下载的商户证书放到项目目录：

```bash
mkdir -p certs
cp /path/to/apiclient_key.pem certs/
chmod 600 certs/apiclient_key.pem
```

### 2. 配置环境变量

编辑 `.env` 文件：

```bash
# 微信支付配置
WECHAT_MCHID=1234567890
WECHAT_SERIAL_NO=ABC123DEF456...
WECHAT_API_V3_KEY=your_api_v3_key_32_characters
WECHAT_PRIVATE_KEY_PATH=./certs/apiclient_key.pem
```

### 3. 插入渠道配置到数据库

连接数据库：

```bash
docker-compose exec postgres psql -U gopay -d gopay
```

插入配置：

```sql
-- 插入微信支付配置
INSERT INTO channel_configs (app_id, channel, config, status) VALUES (
    'test_app_001',
    'wechat_native',
    '{
        "mch_id": "1234567890",
        "serial_no": "ABC123DEF456...",
        "api_v3_key": "your_api_v3_key_32_characters",
        "private_key_path": "./certs/apiclient_key.pem"
    }'::jsonb,
    'active'
);
```

### 4. 验证配置

启动服务：

```bash
go run cmd/gopay/main.go
```

查看日志，应该看到：

```
[INFO] Wechat pay provider initialized successfully, mchID=1234567890
[INFO] Channel provider loaded: appID=test_app_001, channel=wechat_native
```

## 配置说明

### channel_configs 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| app_id | VARCHAR(64) | 应用ID（关联 apps 表） |
| channel | VARCHAR(32) | 支付渠道：wechat_native |
| config | JSONB | 渠道配置（JSON格式） |
| status | VARCHAR(20) | 状态：active/disabled |

### 微信支付配置字段

```json
{
  "mch_id": "商户号",
  "serial_no": "证书序列号",
  "api_v3_key": "APIv3密钥（32位）",
  "private_key_path": "商户私钥文件路径"
}
```

## 测试支付

### 1. 创建支付订单

```bash
curl -X POST http://localhost:8080/api/v1/checkout \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "test_app_001",
    "out_trade_no": "TEST_ORDER_001",
    "amount": 1,
    "subject": "测试商品",
    "channel": "wechat_native",
    "notify_url": "http://your-domain.com/callback"
  }'
```

响应示例：

```json
{
  "order_id": "ORD_20240101_001",
  "pay_url": "weixin://wxpay/bizpayurl?pr=ABC123",
  "qr_code": "weixin://wxpay/bizpayurl?pr=ABC123"
}
```

### 2. 生成二维码

使用 `pay_url` 生成二维码，用户扫码支付。

### 3. 接收回调

微信支付成功后会回调：

```
POST http://your-domain.com/api/v1/webhook/wechat
```

GoPay 会验证签名、更新订单状态、异步通知业务系统。

## 常见问题

### 1. 证书序列号在哪里查看？

登录微信支付商户平台 → 账户中心 → API安全 → 查看证书 → 证书序列号

### 2. APIv3密钥如何设置？

账户中心 → API安全 → 设置APIv3密钥

**注意：** 密钥必须是32位字符，设置后无法查看，请妥善保管。

### 3. 证书文件格式错误

确保下载的是 `apiclient_key.pem` 文件，不是 `apiclient_cert.pem`。

文件内容应该以 `-----BEGIN PRIVATE KEY-----` 开头。

### 4. 回调地址配置

微信支付商户平台 → 产品中心 → 开发配置 → 支付配置

回调地址格式：`https://your-domain.com/api/v1/webhook/wechat`

**注意：** 必须是 HTTPS，不能是 IP 地址。

### 5. 测试环境配置

开发环境可以使用微信支付提供的沙箱环境：

- 沙箱商户号：通过微信支付商户平台申请
- 沙箱密钥：单独设置
- 沙箱API地址：`https://api.mch.weixin.qq.com/sandboxnew/`

## 安全建议

### 1. 证书文件权限

```bash
chmod 600 certs/apiclient_key.pem
```

### 2. 不要提交证书到 Git

`.gitignore` 已包含：

```
certs/
*.pem
*.key
```

### 3. APIv3密钥管理

- 定期更换（建议每年更换一次）
- 不要硬编码在代码中
- 使用环境变量或密钥管理服务

### 4. 回调地址验证

生产环境必须使用 HTTPS，并验证微信支付的签名。

## 多商户号配置

如果有多个微信支付商户号，可以为不同的 app_id 配置不同的商户号：

```sql
-- 应用1使用商户号A
INSERT INTO channel_configs (app_id, channel, config, status) VALUES (
    'app_001',
    'wechat_native',
    '{"mch_id": "1234567890", ...}'::jsonb,
    'active'
);

-- 应用2使用商户号B
INSERT INTO channel_configs (app_id, channel, config, status) VALUES (
    'app_002',
    'wechat_native',
    '{"mch_id": "0987654321", ...}'::jsonb,
    'active'
);
```

```bash

systemctl restart gopay
```
