## API 测试用例

### 1. JSAPI 支付测试

**场景**: 公众号/小程序内支付

**请求示例**:
```bash
curl -X POST http://localhost:8080/api/v1/checkout \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "your_app_id",
    "out_trade_no": "TEST_JSAPI_001",
    "amount": 100,
    "subject": "测试商品-JSAPI",
    "body": "这是一个JSAPI支付测试订单",
    "channel": "wechat_jsapi",
    "notify_url": "https://your-domain.com/webhook/wechat",
    "extra_data": {
      "app_id": "wx1234567890abcdef",
```

## 参数说明

### ExtraData 参数对比

| 参数 | Native | JSAPI | H5 | APP | 说明 |
|-----|--------|-------|----|----|------|
| `app_id` | ✅ | ✅ | ✅ | ✅ | 微信应用 ID（必需） |
| `openid` | ❌ | ✅ | ❌ | ❌ | 用户 openid（JSAPI 必需） |
| `client_ip` | ❌ | ❌ | ✅ | ❌ | 用户 IP（H5 可选，默认 127.0.0.1） |
| `h5_type` | ❌ | ❌ | ✅ | ❌ | H5 类型：Wap/iOS/Android（可选，默认 Wap） |
| `device_id` | ❌ | ❌ | ❌ | ✅ | 设备 ID（APP 可选） |

### 响应字段对比

| 字段 | Native | JSAPI | H5 | APP | 说明 |
|-----|--------|-------|----|----|------|
| `order_no` | ✅ | ✅ | ✅ | ✅ | GoPay 订单号 |
| `pay_url` | ✅ | ❌ | ✅ | ❌ | 支付链接 |
| `qr_code` | ✅ | ❌ | ❌ | ❌ | 二维码内容 |
| `prepay_id` | ❌ | ✅ | ❌ | ✅ | 预支付 ID |
| `pay_info` | ❌ | ✅ | ❌ | ✅ | 调起支付参数 |

## 常见问题

### 1. JSAPI 支付提示 "openid is required"
**原因**: 未传递用户的 openid
**解决**: 在 `extra_data` 中添加 `openid` 参数

### 2. H5 支付跳转后提示 "商家参数格式有误"
**原因**: `client_ip` 或 `h5_type` 参数不正确
**解决**: 确保传递真实的用户 IP，`h5_type` 使用 Wap/iOS/Android

### 3. APP 支付调起失败
**原因**: APP 端未正确集成微信 SDK 或参数传递错误
**解决**: 检查 APP 端的微信 SDK 集成，确保所有参数正确传递

### 4. 回调通知未收到
**原因**: 回调 URL 不可访问或签名验证失败
**解决**: 确保回调 URL 可公网访问，检查证书配置是否正确
