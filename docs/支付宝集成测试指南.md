## API 测试用例

### 1. 扫码支付测试

**场景**: PC 网站扫码支付

**请求示例**:
```bash
curl -X POST http://localhost:8080/api/v1/checkout \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "your_app_id",
    "out_trade_no": "TEST_ALIPAY_QR_001",
    "amount": 100,
    "subject": "测试商品-扫码支付",
    "body": "这是一个支付宝扫码支付测试订单",
    "channel": "alipay_qr",
    "notify_url": "https://your-domain.com/webhook/alipay"
  }'
```

**预期响应**:
```json
{
  "code": 0,
  "message": "订单创建成功",
  "data": {
    "order_no": "GO20260415123456789",
    "pay_url": "https://qr.alipay.com/bax01234567890",
    "qr_code": "https://qr.alipay.com/bax01234567890"
  }
}
```

**前端调用示例**:
```javascript
// 生成二维码展示给用户
import QRCode from 'qrcode';

QRCode.toCanvas(document.getElementById('qrcode'), response.data.qr_code, {
  width: 300
});
```

```java

PayTask alipay = new PayTask(this);
Map<String, String> result = alipay.payV2(orderString, true);
Log.d("Alipay", "支付结果: " + result.toString());
```

## 参数说明

### ExtraData 参数对比

| 参数 | QR | Wap | APP | Face | 说明 |
|-----|----|----|-----|------|------|
| `return_url` | ❌ | ✅ | ❌ | ❌ | 支付成功后跳转地址（Wap 可选） |
| `quit_url` | ❌ | ✅ | ❌ | ❌ | 用户取消支付跳转地址（Wap 可选） |
| `auth_code` | ❌ | ❌ | ❌ | ✅ | 用户付款码（Face 必需） |

### 响应字段对比

| 字段 | QR | Wap | APP | Face | 说明 |
|-----|----|----|-----|------|------|
| `order_no` | ✅ | ✅ | ✅ | ✅ | GoPay 订单号 |
| `pay_url` | ✅ | ✅ | ❌ | ❌ | 支付链接 |
| `qr_code` | ✅ | ❌ | ❌ | ❌ | 二维码内容 |
| `prepay_id` | ❌ | ❌ | ✅ | ❌ | 预支付 ID |
| `pay_info` | ❌ | ❌ | ✅ | ✅ | 调起支付参数 |

## 回调通知

所有支付方式使用相同的回调处理逻辑，无需额外配置。

**回调 URL**: `https://your-domain.com/webhook/alipay`

**回调处理**: 系统会自动验证签名、解析通知、更新订单状态、通知商户。

## 微信 vs 支付宝对比

### 配置复杂度

| 项目 | 微信 | 支付宝 |
|-----|------|--------|
| **商户标识** | mch_id | app_id |
| **签名方式** | RSA + 证书 | RSA |
| **证书管理** | 需要商户证书 + 平台证书 | 只需应用私钥 + 支付宝公钥 |
| **配置复杂度** | 高 | 中 |

### 金额单位

| 项目 | 微信 | 支付宝 |
|-----|------|--------|
| **API 金额单位** | 分 | 元 |
| **存储单位** | 分 | 分（统一转换） |
| **示例** | 100 = 1元 | "1.00" = 1元 |

### 支付方式对比

| 场景 | 微信 | 支付宝 |
|-----|------|--------|
| **PC 扫码** | Native | QR Code |
| **手机网页** | H5 | Wap |
| **APP 支付** | APP | APP |
| **特有方式** | JSAPI（公众号/小程序） | Face（当面付） |

## 常见问题

### 1. 扫码支付提示 "系统繁忙"
**原因**: 配置错误或签名失败
**解决**: 检查 app_id、private_key、alipay_public_key 是否正确

### 2. 手机网站支付跳转后提示 "参数错误"
**原因**: return_url 或 quit_url 格式不正确
**解决**: 确保 URL 是完整的 HTTPS 地址

### 3. APP 支付调起失败
**原因**: APP 端未正确集成支付宝 SDK 或参数传递错误
**解决**: 检查 APP 端的支付宝 SDK 集成，确保 order_string 正确传递

### 4. 当面付提示 "auth_code 无效"
**原因**: 付款码已过期或格式错误
**解决**: 确保使用最新的付款码，付款码有效期为 3 分钟

### 5. 回调通知未收到
**原因**: 回调 URL 不可访问或签名验证失败
**解决**: 确保回调 URL 可公网访问，检查支付宝公钥配置是否正确
