# 密钥轮换指南

## 概述

本文档描述 GoPay 支付网关的密钥轮换机制和操作流程。

## 支持的密钥类型

1. **支付渠道密钥**
   - 支付宝应用私钥 (ALIPAY_APP_PRIVATE_KEY)
   - 微信支付 API v3 密钥 (WECHAT_API_V3_KEY)
   - 微信支付证书 (WECHAT_CERT_PATH, WECHAT_KEY_PATH)

2. **管理后台密钥**
   - 管理后台 API Key (ADMIN_API_KEY)

3. **数据库密钥**
   - 数据库密码 (DB_PASSWORD)

## 轮换策略

### 自动轮换
- 管理后台 API Key：每 90 天自动轮换
- 数据库密码：建议每 180 天轮换

### 手动轮换
- 支付渠道密钥：根据支付平台要求或安全事件触发
- 发生安全事件时立即轮换所有密钥

## 轮换流程

### 1. 管理后台 API Key 轮换

```bash
# 生成新的 API Key
go run cmd/rotate-keys/main.go --type=admin-api-key

# 更新环境变量
export ADMIN_API_KEY="新生成的密钥"

# 重启服务
podman compose --env-file .env restart gopay

# 验证新密钥
curl -H "X-API-Key: 新密钥" http://localhost:8080/admin/health
```

### 2. 支付宝密钥轮换

```bash
# 1. 在支付宝开放平台生成新的应用私钥
# 2. 上传新的应用公钥到支付宝平台
# 3. 更新环境变量
export ALIPAY_APP_PRIVATE_KEY="新的私钥内容"

# 4. 使用零停机轮换
go run cmd/rotate-keys/main.go --type=alipay --verify

# 5. 重启服务
podman compose --env-file .env restart gopay
```

### 3. 微信支付密钥轮换

```bash
# 1. 在微信支付商户平台生成新的 API v3 密钥
# 2. 更新环境变量
export WECHAT_API_V3_KEY="新的密钥"

# 3. 下载新的商户证书
# 4. 更新证书路径
export WECHAT_CERT_PATH="/path/to/new/cert.pem"
export WECHAT_KEY_PATH="/path/to/new/key.pem"

# 5. 验证并重启
go run cmd/rotate-keys/main.go --type=wechat --verify
podman compose --env-file .env restart gopay
```

### 4. 数据库密码轮换

```bash
# 1. 在数据库中创建新密码
psql -U postgres -c "ALTER USER gopay WITH PASSWORD '新密码';"

# 2. 更新环境变量
export DB_PASSWORD="新密码"

# 3. 重启服务（会自动使用新密码）
podman compose --env-file .env restart gopay

# 4. 验证连接
go run cmd/rotate-keys/main.go --type=db --verify
```

## 密钥存储最佳实践

### 生产环境
- 使用 Kubernetes Secrets 或 AWS Secrets Manager
- 启用密钥版本控制
- 设置密钥访问审计日志

### 开发环境
- 使用 .env 文件（不提交到 Git）
- 使用测试密钥，不使用生产密钥

## 应急响应

### 密钥泄露处理流程

1. **立即响应**（5 分钟内）
   ```bash
   # 禁用泄露的密钥
   go run cmd/rotate-keys/main.go --type=all --emergency
   ```

2. **生成新密钥**（15 分钟内）
   - 生成所有新密钥
   - 更新支付平台配置

3. **部署更新**（30 分钟内）
   - 更新生产环境密钥
   - 重启所有服务实例

4. **验证和监控**（持续）
   - 验证所有支付功能
   - 监控异常交易
   - 审查访问日志

## 自动化脚本

### 定期轮换提醒

在 crontab 中添加：
```bash
# 每月 1 号检查密钥年龄
0 0 1 * * /app/scripts/check-key-age.sh
```

### 密钥年龄监控

```bash
# 检查密钥年龄
go run cmd/rotate-keys/main.go --check-age

# 输出示例：
# ADMIN_API_KEY: 45 天（建议 90 天轮换）
# ALIPAY_APP_PRIVATE_KEY: 120 天（警告：超过 90 天）
# WECHAT_API_V3_KEY: 30 天
```

## 合规要求

- PCI DSS：密钥至少每 90 天轮换一次
- 支付宝/微信支付：遵循平台密钥管理规范
- 内部审计：记录所有密钥轮换操作

## 故障排查

### 轮换后支付失败

1. 检查新密钥格式是否正确
2. 验证支付平台是否已更新公钥
3. 检查证书有效期
4. 查看应用日志中的签名错误

### 服务无法启动

1. 验证环境变量是否正确设置
2. 检查密钥文件权限
3. 验证数据库连接
