## 已落地的关键能力

### 1. 部署与迁移

- 统一使用 `golang-migrate` 作为正式迁移入口
- 运行时和命令行迁移入口一致
- `002_indexes` 已纳入正式迁移版本
- `docker-compose.prod.yml` 和 Helm 模板可用于生产部署
- 本地开发与生产容器命令已统一为 `podman compose`

### 2. 支付主链路

- 支付宝表单回调可正确验签、解析和更新订单
- 微信配置字段与运行时一致，支持私钥内容与私钥文件路径
- 微信退款请求使用原始订单总金额，避免部分退款参数错误
- `app_id + out_trade_no` 已具备数据库级唯一约束

### 3. 安全与稳定性

- 生产环境必须显式设置 `SERVER_ENV`、`ADMIN_API_KEY`、`PUBLIC_BASE_URL`
- 管理后台 IP 白名单支持 CIDR
- Redis 限流检查失败时不再默认放行
- 本地限流真正按 `Rate/Burst` 生效
- 异步通知在 worker 池满时不再静默丢失，并有定时补偿任务兜底
- 渠道配置敏感字段会加密入库、脱敏审计、脱敏返回

### 4. 管理后台与运维

- 失败订单、通知失败、订单详情查询已对齐真实 schema
- 对账差异类型与数据库枚举值一致
- 管理后台重试会重置状态，后续由补偿任务重新拉起通知
- 部署检查文档、项目状态文档与当前代码路径一致

---

## 当前仍建议关注的非阻塞事项

- 真实支付渠道沙箱和生产环境应各跑一轮端到端支付回归
- Helm / Compose 的上线参数建议由 CI 或密钥管理系统统一注入
- 对账和通知补偿任务建议在生产监控中单独加告警
