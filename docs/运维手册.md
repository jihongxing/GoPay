# GoPay 运维手册

## 1. 服务管理

### 启动服务

```bash
# Podman Compose（推荐）
podman compose --env-file .env up -d

# 直接运行
go run cmd/gopay/main.go

# 生产环境
podman compose --env-file .env.prod -f docker-compose.prod.yml up -d
```

### 停止服务

```bash
podman compose --env-file .env down
```

### 查看日志

```bash
# 实时日志
podman compose --env-file .env logs -f gopay

# 最近 100 行
podman compose --env-file .env logs --tail=100 gopay
```

### 健康检查

```bash
# 基础健康检查
curl http://localhost:8080/health

# 详细健康检查（含数据库状态）
curl http://localhost:8080/health/detail
```

## 2. 数据库运维

### 连接数据库

```bash
psql -h localhost -U gopay -d gopay
```

### 数据库迁移

```bash
# 运行迁移
go run cmd/migrate/main.go

# 手动执行
psql -h localhost -U gopay -d gopay -f migrations/001_init_schema.up.sql
```

### 常用查询

```sql
-- 查看失败订单
SELECT order_no, app_id, channel, amount, status, notify_status, retry_count, created_at
FROM orders
WHERE status = 'paid' AND notify_status = 'failed_notify'
ORDER BY created_at DESC
LIMIT 20;

-- 查看今日订单统计
SELECT channel, status, COUNT(*), SUM(amount) as total_amount
FROM orders
WHERE created_at >= CURRENT_DATE
GROUP BY channel, status;

-- 查看通知失败率
SELECT
  COUNT(*) FILTER (WHERE notify_status = 'failed_notify') as failed,
  COUNT(*) FILTER (WHERE notify_status = 'notified') as success,
  COUNT(*) as total
FROM orders
WHERE created_at >= CURRENT_DATE;
```

### 数据库备份

```bash
# 全量备份
pg_dump -h localhost -U gopay gopay > backup_$(date +%Y%m%d).sql

# 仅备份订单表
pg_dump -h localhost -U gopay -t orders gopay > orders_backup_$(date +%Y%m%d).sql

# 恢复
psql -h localhost -U gopay gopay < backup_20260419.sql
```

## 3. 监控

### Prometheus 指标

访问 `http://localhost:8080/metrics` 查看所有指标。

关键指标：
- `gopay_http_requests_total` — HTTP 请求总数
- `gopay_http_request_duration_seconds` — 请求延迟
- `gopay_payment_total` — 支付请求数（按渠道、状态）
- `gopay_notify_total` — 通知请求数（按状态）

### Grafana Dashboard

访问 `http://localhost:3000`，默认账号 admin/admin。

预置 Dashboard：
- System Overview — 系统概览（QPS、延迟、错误率）
- Business Monitoring — 业务监控（订单量、成功率、渠道分布）

### 告警规则

告警通过 Webhook 发送，配置 `ALERT_WEBHOOK_URL` 环境变量。

告警场景：
- 支付失败率 > 5%
- 通知重试率过高
- 对账差异
- 证书即将过期（提前 30 天）
- 系统异常

## 4. 证书管理

### 微信支付证书

证书文件存放在 `certs/wechat/` 目录：
- `apiclient_key.pem` — 商户私钥
- `apiclient_cert.pem` — 商户证书

### 证书更新流程

1. 从微信商户平台下载新证书
2. 替换 `certs/wechat/` 下的文件
3. 重启服务：`podman compose --env-file .env restart gopay`
4. 验证：检查日志中无证书错误

### 证书过期检查

系统启动时自动检查证书有效期，每天定期检查。过期前 30 天发送告警。

手动检查：
```bash
openssl x509 -in certs/wechat/apiclient_cert.pem -noout -dates
```

## 7. 容量规划

### 推荐配置

| 日订单量 | CPU | 内存 | 数据库 |
|---------|-----|------|--------|
| < 1000 | 1 核 | 512MB | 1 核 1GB |
| 1000-10000 | 2 核 | 1GB | 2 核 2GB |
| > 10000 | 4 核 | 2GB | 4 核 4GB |

### 数据库连接池

默认配置适用于大多数场景。高并发时调整：

```bash
DB_MAX_OPEN_CONNS=50    # 最大打开连接数
DB_MAX_IDLE_CONNS=10    # 最大空闲连接数
DB_CONN_MAX_LIFETIME=1h # 连接最大生命周期
```

## 9. 性能优化

### 9.1 数据库优化

**索引优化**
```sql
-- 检查缺失的索引
SELECT schemaname, tablename, attname, n_distinct, correlation
FROM pg_stats
WHERE schemaname = 'public' AND tablename = 'orders'
ORDER BY abs(correlation) DESC;

-- 查看索引使用情况
SELECT schemaname, tablename, indexname, idx_scan, idx_tup_read, idx_tup_fetch
FROM pg_stat_user_indexes
WHERE schemaname = 'public'
ORDER BY idx_scan ASC;

-- 查找未使用的索引
SELECT schemaname, tablename, indexname
FROM pg_stat_user_indexes
WHERE idx_scan = 0 AND schemaname = 'public';
```

**查询优化**
```sql
-- 分析慢查询
EXPLAIN ANALYZE
SELECT * FROM orders WHERE app_id = 'test_app' AND status = 'paid';

-- 更新表统计信息
ANALYZE orders;
ANALYZE transactions;

-- 清理死元组
VACUUM ANALYZE orders;
```

**连接池调优**
```go
// 在 internal/database/database.go 中调整
db.SetMaxOpenConns(100)        // 根据并发量调整
db.SetMaxIdleConns(50)         // 通常为 MaxOpenConns 的 50%
db.SetConnMaxLifetime(time.Hour) // 避免长连接问题
db.SetConnMaxIdleTime(10 * time.Minute)
```

### 9.2 应用层优化

**Worker Pool 调优**
```go
// 在 internal/service/notify_service.go 中
workerPool := pool.NewWorkerPool(200) // 根据 CPU 核心数调整
```

**HTTP 客户端优化**
```go
// 复用 HTTP 连接
var httpClient = &http.Client{
    Timeout: 3 * time.Second,
    Transport: &http.Transport{
        MaxIdleConns:        100,
        MaxIdleConnsPerHost: 10,
        IdleConnTimeout:     90 * time.Second,
    },
}
```

### 9.3 系统层优化

**文件描述符限制**
```bash
# 查看当前限制
ulimit -n

# 临时提高限制
ulimit -n 65535

# 永久修改（编辑 /etc/security/limits.conf）
* soft nofile 65535
* hard nofile 65535
```

**TCP 参数优化**
```bash
# 编辑 /etc/sysctl.conf
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 2048
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30

# 应用配置
sysctl -p
```

## 10. 日志管理

### 10.1 日志级别

通过环境变量 `LOG_LEVEL` 控制：
- `debug` — 调试信息（开发环境）
- `info` — 一般信息（默认）
- `warn` — 警告信息
- `error` — 错误信息（生产环境推荐）

### 10.2 日志查看

```bash
# 查看实时日志
podman compose --env-file .env logs -f gopay

# 查看错误日志
podman compose --env-file .env logs gopay | grep ERROR

# 查看特定订单的日志
podman compose --env-file .env logs gopay | grep "ORDER_20260419_001"

# 查看支付失败日志
podman compose --env-file .env logs gopay | grep "payment failed"
```

### 10.3 日志轮转

使用 logrotate 管理日志文件：

```bash
# 创建 /etc/logrotate.d/gopay
/var/log/gopay/*.log {
    daily
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 gopay gopay
    sharedscripts
    postrotate
        podman compose --env-file .env restart gopay
    endscript
}
```

### 10.4 日志分析

```bash
# 统计今日订单数
grep "Creating order" gopay.log | grep "$(date +%Y-%m-%d)" | wc -l

# 统计各渠道订单数
grep "Creating order" gopay.log | grep -oP 'channel=\w+' | sort | uniq -c

# 统计错误类型
grep "ERROR" gopay.log | grep -oP 'error=\w+' | sort | uniq -c

# 查找慢请求（响应时间 > 1s）
grep "duration" gopay.log | awk '$NF > 1000' | tail -20
```

## 11. 安全加固

### 11.1 网络安全

**防火墙配置**
```bash
# 只允许特定 IP 访问管理接口
iptables -A INPUT -p tcp --dport 8080 -s 192.168.1.0/24 -j ACCEPT
iptables -A INPUT -p tcp --dport 8080 -j DROP

# 使用 ufw（Ubuntu）
ufw allow from 192.168.1.0/24 to any port 8080
ufw enable
```

**HTTPS 配置**
```nginx
# Nginx 反向代理配置
server {
    listen 443 ssl http2;
    server_name pay.example.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 11.2 访问控制

**管理后台 IP 白名单**
```bash
# 在 .env 中配置
ADMIN_IP_WHITELIST=127.0.0.1,192.168.1.100,10.0.0.0/8
```

**API Key 管理**
```bash
# 生成强随机 API Key
openssl rand -base64 32

# 定期轮换 API Key（建议每 90 天）
# 使用密钥轮换工具
go run cmd/rotate-keys/main.go
```

### 11.3 数据安全

**敏感数据加密**
- 商户密钥使用 AES-256-GCM 加密存储
- MASTER_KEY 通过环境变量注入，不写入代码
- 日志自动脱敏（手机号、身份证、银行卡）

**数据库安全**
```bash
# 使用强密码
DB_PASSWORD=$(openssl rand -base64 32)

# 限制数据库访问
# 编辑 postgresql.conf
listen_addresses = 'localhost'

# 编辑 pg_hba.conf
host    gopay    gopay    127.0.0.1/32    md5
```

## 12. 灾难恢复

### 12.1 备份策略

**自动备份脚本**
```bash
#!/bin/bash
# /usr/local/bin/gopay-backup.sh

BACKUP_DIR="/backup/gopay"
DATE=$(date +%Y%m%d_%H%M%S)

# 数据库备份
pg_dump -h localhost -U gopay gopay | gzip > "$BACKUP_DIR/db_$DATE.sql.gz"

# 配置文件备份
tar -czf "$BACKUP_DIR/config_$DATE.tar.gz" /etc/gopay/

# 证书备份
tar -czf "$BACKUP_DIR/certs_$DATE.tar.gz" /etc/gopay/certs/

# 删除 30 天前的备份
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete

echo "Backup completed: $DATE"
```

**定时任务**
```bash
# 添加到 crontab
0 2 * * * /usr/local/bin/gopay-backup.sh >> /var/log/gopay-backup.log 2>&1
```

### 12.2 恢复流程

**数据库恢复**
```bash
# 1. 停止服务
podman compose --env-file .env stop gopay

# 2. 恢复数据库
gunzip < db_20260419_020000.sql.gz | psql -h localhost -U gopay gopay

# 3. 验证数据
psql -h localhost -U gopay gopay -c "SELECT COUNT(*) FROM orders;"

# 4. 启动服务
podman compose --env-file .env start gopay

# 5. 验证服务
curl http://localhost:8080/health
```

**配置恢复**
```bash
# 恢复配置文件
tar -xzf config_20260419_020000.tar.gz -C /

# 恢复证书
tar -xzf certs_20260419_020000.tar.gz -C /

# 重启服务
podman compose --env-file .env restart gopay
```

### 12.3 故障切换

**主备切换流程**
1. 确认主节点故障
2. 停止主节点服务
3. 将备节点数据库提升为主库
4. 更新 DNS 或负载均衡配置
5. 启动备节点服务
6. 验证服务正常

**数据库主备切换**
```bash
# 在备库执行
pg_ctl promote -D /var/lib/postgresql/data

# 更新应用配置
DB_HOST=backup-db-host

# 重启应用
podman compose --env-file .env restart gopay
```

**升级前检查**
```bash
# 1. 备份数据
/usr/local/bin/gopay-backup.sh

# 2. 查看当前版本
curl http://localhost:8080/version
```

```bash
podman compose --env-file .env down
```

```bash
# 3. 恢复数据库（如果有必要）

```

**计划维护**
1. 提前通知业务方（至少提前 24 小时）

## 14. 监控告警

### 14.1 关键指标

**系统指标**
- CPU 使用率 < 80%
- 内存使用率 < 85%
- 磁盘使用率 < 80%
- 网络带宽使用率 < 70%

**应用指标**
- QPS（每秒请求数）
- 响应时间（P95 < 100ms）
- 错误率 < 0.1%
- 数据库连接数 < 最大连接数的 80%

**业务指标**
- 支付成功率 > 99%
- 通知成功率 > 99.5%
- 对账差异数 = 0
- 退款成功率 > 99%

### 14.2 告警配置

**Prometheus 告警规则**
```yaml
# prometheus/alerts.yml
groups:
  - name: gopay
    interval: 30s
    rules:
      - alert: HighErrorRate
        expr: rate(gopay_http_requests_total{status=~"5.."}[5m]) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "高错误率告警"
          description: "错误率超过 1%"

      - alert: HighResponseTime
        expr: histogram_quantile(0.95, gopay_http_request_duration_seconds_bucket) > 0.1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "响应时间过长"
          description: "P95 响应时间超过 100ms"

      - alert: DatabaseConnectionPoolExhausted
        expr: gopay_db_connections_in_use / gopay_db_connections_max > 0.9
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "数据库连接池即将耗尽"
          description: "连接池使用率超过 90%"
```

### 14.3 告警处理

**告警响应流程**
1. 接收告警通知（钉钉/邮件/短信）
2. 确认告警真实性
3. 评估影响范围
4. 执行应急预案
5. 记录处理过程
6. 事后复盘

**常见告警处理**

| 告警类型 | 处理方法 |
|---------|---------|
| 高错误率 | 查看错误日志，定位问题，必要时回滚 |
| 响应时间长 | 检查数据库慢查询，优化或扩容 |
| 连接池耗尽 | 增加连接池大小，检查连接泄漏 |
| 磁盘空间不足 | 清理日志，扩容磁盘 |
| 证书即将过期 | 更新证书，重启服务 |
| 对账差异 | 人工核对，联系支付平台 |
