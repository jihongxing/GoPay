## 前置准备

### 1. 申请接入

向 GoPay 管理员提供以下信息：

- **业务系统名称**: 例如 "电商系统"、"会员系统"
- **业务描述**: 简要说明业务场景
- **需要的支付渠道**: 
  - 微信支付：`wechat_native`（扫码）、`wechat_jsapi`（公众号/小程序）、`wechat_h5`（H5）、`wechat_app`（APP）
  - 支付宝：`alipay_qr`（扫码）、`alipay_wap`（手机网站）、`alipay_app`（APP）、`alipay_face`（当面付）
- **回调地址**: 接收支付结果通知的 URL，例如 `https://your-domain.com/payment/callback`
- **技术对接人**: 姓名、邮箱、电话

### 2. 获取凭证

管理员审核通过后，会提供：

- **AppID**: 应用唯一标识，例如 `app_20260416001`
- **AppSecret**: 应用密钥，用于签名验证（请妥善保管，不要泄露）
- **GoPay 网关地址**: 
  - 沙箱环境: `https://gopay-sandbox.your-domain.com`
  - 生产环境: `https://gopay.your-domain.com`

## 创建应用

管理员可通过以下方式创建应用：

### 方式 1: 管理后台（推荐）

1. 访问管理后台: `http://gopay-host:8080/admin`
2. 点击"应用管理" → "创建应用"
3. 填写应用信息：
   - 应用名称
   - 回调地址
   - 选择支付渠道
4. 保存后获取 AppID 和 AppSecret

### 方式 2: API 接口

```bash
curl -X POST http://gopay-host:8080/admin/api/apps \
  -H "Content-Type: application/json" \
  -H "X-Admin-API-Key: your-admin-api-key" \
  -d '{
    "name": "电商系统",
    "callback_url": "https://your-business.com/payment/callback",
    "channels": ["alipay_qr", "wechat_native"]
  }'
```

响应示例：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "app_id": "app_20260416001",
    "app_secret": "sk_live_xxxxxxxxxxxxxxxx",
    "name": "电商系统",
    "callback_url": "https://your-business.com/payment/callback",
    "status": "active",
    "created_at": "2026-04-16T10:00:00Z"
  }
}
```

- **Base URL**: `https://gopay.your-domain.com`

## 签名算法

### 签名生成步骤

1. **构造待签名字符串**

```
待签名字符串格式：
{timestamp}\n{nonce}\n{request_body}\n
```

示例：
```
1713254400
abc123def456
{"out_trade_no":"ORDER_20260416001","amount":10000,"subject":"测试商品","channel":"wechat_native"}

```

2. **计算签名**

使用 HMAC-SHA256 算法，密钥为 AppSecret：

```go
// Go 示例
import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
)

func calculateSignature(timestamp, nonce, body, appSecret string) string {
    message := fmt.Sprintf("%s\n%s\n%s\n", timestamp, nonce, body)
    h := hmac.New(sha256.New, []byte(appSecret))
    h.Write([]byte(message))
    return hex.EncodeToString(h.Sum(nil))
}
```

```python
# Python 示例
import hmac
import hashlib

def calculate_signature(timestamp, nonce, body, app_secret):
    message = f"{timestamp}\n{nonce}\n{body}\n"
    signature = hmac.new(
        app_secret.encode('utf-8'),
        message.encode('utf-8'),
        hashlib.sha256
    ).hexdigest()
    return signature
```

```javascript
// Node.js 示例
const crypto = require('crypto');

function calculateSignature(timestamp, nonce, body, appSecret) {
    const message = `${timestamp}\n${nonce}\n${body}\n`;
    const hmac = crypto.createHmac('sha256', appSecret);
    hmac.update(message);
    return hmac.digest('hex');
}
```

3. **设置请求头**

```
X-App-ID: app_20260416001
X-Timestamp: 1713254400
X-Nonce: abc123def456
X-Signature: calculated_signature_hex
```

### 签名验证

GoPay 网关会验证以下内容：

1. **时间戳验证**: 请求时间戳与服务器时间差不超过 5 分钟
2. **随机数验证**: 防止重放攻击（建议使用 UUID）
3. **签名验证**: 使用 AppSecret 验证签名是否正确

## 回调处理

### 回调机制

- **触发时机**: 支付成功后，GoPay 会向业务系统的回调地址发送通知
- **重试策略**: 如果回调失败，会按照以下间隔重试：
  - 第 1 次: 立即
  - 第 2 次: 5 秒后
  - 第 3 次: 30 秒后
  - 第 4 次: 2 分钟后
  - 第 5 次: 10 分钟后
  - 第 6 次: 30 分钟后
  - 第 7 次: 1 小时后
  - 第 8 次: 2 小时后
- **超时时间**: 3 秒
- **成功标识**: 业务系统返回 HTTP 200 且响应体为 `SUCCESS`

### 回调请求格式

**请求方式**: `POST`

**请求头**:
```
Content-Type: application/json
X-GoPay-Signature: callback_signature
```

**请求体**:
```json
{
  "order_no": "GO20260416123456789",
  "out_trade_no": "ORDER_20260416001",
  "amount": 10000,
  "channel": "wechat_native",
  "status": "success",
  "paid_at": "2026-04-16T10:05:00Z",
  "extra_data": {
    "user_id": "12345",
    "order_type": "product"
  }
}
```

### 回调签名验证

1. **提取签名**: 从请求头 `X-GoPay-Signature` 获取签名
2. **构造待验证字符串**: 使用请求体原始 JSON 字符串
3. **验证签名**:

```go
// Go 示例
func verifyCallbackSignature(body []byte, signature, appSecret string) bool {
    h := hmac.New(sha256.New, []byte(appSecret))
    h.Write(body)
    expected := hex.EncodeToString(h.Sum(nil))
    return hmac.Equal([]byte(signature), []byte(expected))
}
```

```python
# Python 示例
import hmac
import hashlib

def verify_callback_signature(body, signature, app_secret):
    expected = hmac.new(
        app_secret.encode('utf-8'),
        body.encode('utf-8'),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, expected)
```

### 回调处理示例

```go
// Go 示例
func PaymentCallbackHandler(w http.ResponseWriter, r *http.Request) {
    // 1. 读取请求体
    body, err := io.ReadAll(r.Body)
    if err != nil {
        w.WriteHeader(http.StatusBadRequest)
        return
    }
    
    // 2. 验证签名
    signature := r.Header.Get("X-GoPay-Signature")
    if !verifyCallbackSignature(body, signature, appSecret) {
        w.WriteHeader(http.StatusUnauthorized)
        return
    }
    
    // 3. 解析通知数据
    var notify struct {
        OrderNo     string                 `json:"order_no"`
        OutTradeNo  string                 `json:"out_trade_no"`
        Amount      int64                  `json:"amount"`
        Channel     string                 `json:"channel"`
        Status      string                 `json:"status"`
        PaidAt      string                 `json:"paid_at"`
        ExtraData   map[string]interface{} `json:"extra_data"`
    }
    if err := json.Unmarshal(body, &notify); err != nil {
        w.WriteHeader(http.StatusBadRequest)
        return
    }
    
    // 4. 幂等性检查（防止重复处理）
    if isOrderProcessed(notify.OutTradeNo) {
        w.Write([]byte("SUCCESS"))
        return
    }
    
    // 5. 更新业务订单状态
    if err := updateOrderStatus(notify.OutTradeNo, notify.Status); err != nil {
        w.WriteHeader(http.StatusInternalServerError)
        return
    }
    
    // 6. 返回成功响应
    w.WriteHeader(http.StatusOK)
    w.Write([]byte("SUCCESS"))
}
```

### 重要提示

⚠️ **回调处理注意事项**:

1. **必须验证签名**: 防止恶意请求伪造支付成功通知
2. **必须验证金额**: 确保回调金额与订单金额一致
3. **必须幂等处理**: 同一订单可能收到多次回调，避免重复处理
4. **必须快速响应**: 3 秒内返回结果，避免超时重试
5. **异步处理业务逻辑**: 复杂业务逻辑应放入消息队列异步处理
6. **记录回调日志**: 便于问题排查

## 错误码说明

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 签名验证失败 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 业务错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 1001 | 参数缺失 | 检查必填参数 |
| 1002 | 参数格式错误 | 检查参数类型和格式 |
| 1003 | 签名验证失败 | 检查签名算法和 AppSecret |
| 1004 | 时间戳过期 | 检查系统时间是否同步 |
| 2001 | 应用不存在 | 检查 AppID 是否正确 |
| 2002 | 应用已禁用 | 联系管理员启用应用 |
| 2003 | 渠道未配置 | 联系管理员配置支付渠道 |
| 3001 | 订单已存在 | 使用新的 out_trade_no |
| 3002 | 订单不存在 | 检查订单号是否正确 |
| 3003 | 订单状态异常 | 查询订单状态后重试 |
| 4001 | 支付渠道错误 | 检查渠道参数是否正确 |
| 4002 | 支付金额错误 | 检查金额是否大于 0 |
| 5001 | 系统繁忙 | 稍后重试 |

**错误响应示例**:
```json
{
  "code": 1003,
  "message": "签名验证失败",
  "data": null
}
```

## 常见问题

### Q1: 签名验证失败怎么办？

**A**: 检查以下几点：
1. AppSecret 是否正确
2. 时间戳格式是否正确（Unix 时间戳，秒级）
3. 随机数是否包含在签名中
4. 请求体 JSON 是否与签名时一致（注意空格和换行）
5. 签名算法是否为 HMAC-SHA256

### Q2: 回调没有收到怎么办？

**A**: 检查以下几点：
1. 回调地址是否可公网访问
2. 回调接口是否返回 HTTP 200 和 "SUCCESS"
3. 回调接口是否在 3 秒内响应
4. 查看 GoPay 管理后台的通知日志

### Q3: 订单状态不一致怎么办？

**A**: 
1. 主动调用查询接口获取最新状态
2. 不要仅依赖回调，回调可能延迟或失败
3. 建议定时轮询未支付订单状态

### Q4: 如何处理重复回调？

**A**: 
1. 在数据库中使用唯一索引（out_trade_no）
2. 在更新订单状态前检查当前状态
3. 使用分布式锁防止并发更新

### Q5: 支付金额单位是什么？

**A**: 
- **统一使用"分"作为单位**
- 100.00 元 = 10000 分
- 避免浮点数精度问题

### Q6: 如何测试回调？

**A**: 
1. 使用 ngrok 或 frp 将本地服务暴露到公网
2. 在沙箱环境完成支付后，GoPay 会自动发送回调
3. 使用管理后台的"重发通知"功能手动触发回调

### Q7: 生产环境上线前需要做什么？

**A**: 
1. 完成沙箱环境测试
2. 更换为生产环境的 AppID 和 AppSecret
3. 更新网关地址为生产环境
4. 配置生产环境的回调地址
5. 进行小额真实支付测试
6. 监控日志和告警
