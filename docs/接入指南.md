# GoPay 接入指南

本文档面向需要接入 GoPay 支付网关的业务系统开发者。

## 接入架构

```
┌──────────────┐     HTTPS + 签名      ┌──────────────┐     API     ┌──────────────┐
│   业务系统    │ ──────────────────────→ │    GoPay     │ ──────────→ │  微信/支付宝  │
│  (你的项目)   │ ←────────────────────── │   支付网关    │ ←────────── │   支付平台    │
└──────────────┘    HTTP 回调通知        └──────────────┘   Webhook   └──────────────┘
```

业务系统只需要和 GoPay 交互，不需要直接对接微信/支付宝。

## 第一步：获取凭证

联系 GoPay 管理员，提供以下信息：

| 信息 | 说明 | 示例 |
|------|------|------|
| 应用名称 | 你的业务系统名称 | 酒店竞价系统 |
| 回调地址 | 接收支付/退款通知的 URL | `https://hotel.example.com/payment/callback` |
| 需要的支付渠道 | 微信扫码、支付宝等 | wechat_native, alipay_qr |

管理员会返回给你：

- `app_id` — 应用标识
- `app_secret` — 签名密钥（妥善保管，不要泄露）

## 第二步：实现签名

每次调用 GoPay API 都需要签名。算法：

```
signature = HMAC-SHA256(app_secret, request_body + "\n" + timestamp + "\n" + nonce)
```

请求头：

```
X-App-ID: your_app_id
X-Timestamp: 1713345600
X-Nonce: a1b2c3d4-e5f6-7890
X-Signature: 5d41402abc4b2a76b9719d911017c592...
```

各语言实现见 [API 文档](api/README.md#签名认证) 或 `examples/` 目录。

## 第三步：调用下单接口

```bash
curl -X POST https://pay.example.com/api/v1/checkout \
  -H "Content-Type: application/json" \
  -H "X-App-ID: your_app_id" \
  -H "X-Timestamp: 1713345600" \
  -H "X-Nonce: random-uuid" \
  -H "X-Signature: hmac-sha256-hex" \
  -d '{
    "app_id": "your_app_id",
    "out_trade_no": "ORDER_20260417_001",
    "amount": 100,
    "subject": "测试商品",
    "channel": "wechat_native"
  }'
```

返回：

```json
{
  "code": "SUCCESS",
  "message": "订单创建成功",
  "data": {
    "order_no": "GP20260417143052123456",
    "qr_code": "weixin://wxpay/bizpayurl?pr=abc123"
  }
}
```

前端拿到 `qr_code` 生成二维码展示给用户扫码支付。

## 第四步：实现回调接口

支付成功后 GoPay 会 POST 通知到你的回调地址：

```json
{
  "order_no": "GP20260417143052123456",
  "out_trade_no": "ORDER_20260417_001",
  "amount": 100,
  "status": "paid",
  "channel": "wechat_native",
  "paid_at": "2026-04-17T14:31:00+08:00"
}
```

你的回调接口需要：

1. 根据 `out_trade_no` 查找业务订单
2. 校验金额是否一致
3. 做幂等处理（同一订单不要重复处理）
4. 返回 HTTP 200（3 秒内响应）

Python Flask 示例：

```python
@app.route('/payment/callback', methods=['POST'])
def payment_callback():
    data = request.get_json()

    order = db.get_order(data['out_trade_no'])
    if not order:
        return 'order not found', 400

    if order.status == 'paid':
        return 'ok', 200  # 幂等：已处理过

    if order.amount != data['amount']:
        return 'amount mismatch', 400

    db.update_order_status(order.id, 'paid')
    # 异步处理后续业务（发货、通知用户等）

    return 'ok', 200
```

## 第五步：前端轮询（可选）

用户扫码后，前端可以轮询订单状态：

```javascript
async function pollPayment(orderNo) {
  for (let i = 0; i < 150; i++) {  // 最多 5 分钟
    const resp = await fetch(`/api/v1/orders/${orderNo}`, {
      headers: { /* 签名头 */ }
    });
    const data = await resp.json();

    if (data.data.status === 'paid') {
      showSuccess();
      return;
    }
    await new Promise(r => setTimeout(r, 2000));  // 2 秒间隔
  }
  showTimeout();
}
```

## 常见问题

**Q: 回调没收到怎么办？**
GoPay 会自动重试 5 次（1s, 2s, 4s, 8s, 16s）。如果全部失败，管理员可以在管理后台手动重试。请确保回调地址公网可达且响应时间 < 3 秒。

**Q: 同一个订单号可以重复下单吗？**
不可以。`out_trade_no` 在同一个 `app_id` 下必须唯一，重复提交会返回 `ORDER_EXISTS` 错误。

**Q: 支持部分退款吗？**
支持。退款接口传 `amount` 参数即可指定退款金额，不传则全额退款。

**Q: 签名验证失败怎么排查？**
见 [错误码文档](api/ERROR_CODES.md#签名验证错误排查)。

**Q: 如何区分支付通知和退款通知？**
通过回调请求体中的 `status` 字段：`paid` 是支付通知，`refunded` 是退款通知。
